## Vector Embeddings

- Before data can be used in a vector database, it must be properly loaded and embedded.
    - Embedding is the process of converting data into a format suitable for vector storage.

### Practical Implementation

- For processing large amounts of text, `text-embedding-3-small` is a common choice for setting up vector databases.

### The Purpose of Vector Embeddings

- They help AI and computers understand different types of data being fed into a vector database
- Essentially, embeddings act as a way to represent data as numbers
- This numerical representation allows the system to process various modalities:
    - **Text Embeddings**: Capture meaning (e.g., used in ChatGPT or search engines)
    - **Image Embeddings**: Represent visual features (e.g., used in visual search or product recommenders)
    - **Audio Embeddings**: Capture sound patterns (e.g., used in music recognition)
    - **Video Embeddings**: Capture images and motion over time (e.g., used in action detection)
    - **Multimodal Embeddings**: Combine different data types (e.g., text and images for media recommendations)
    - **Graph Embeddings**: Represent relationships in networks (e.g., social networks)

### How Embeddings Capture Meaning

- Embeddings represent the unique qualities and characteristics of complex data as numbers
- This allows the AI to recognize semantic similarity rather than just superficial similarity:
    - **Text Example**: The word "cat" is mathematically closer to "kitten" than to "car," even though "cat" and "car" look more similar visually.
    - **Image Example**: An image of a beach is represented as being more similar to an image of water than to an image of a mountain.
