### OpenAI Tokenizer

- A tool used to examine how different models process text
    - Users can input prompts to see how they are split into tokens and characters
    - It helps visualize the relationship between raw text and the discrete units (tokens) a model actually processes

### Tokenization Metrics and Rules

- **Token IDs**: The tool displays the specific numerical IDs assigned to each token
- **Rule of Thumb for English Text**:
    - 1 token $\approx$ 4 characters
    - 1 token $\approx$ 0.75 words
    - 100 tokens $\approx$ 75 words
- **Model Variability**
    - Token counts can vary slightly depending on the specific model being used
    - **[Why this matters]** Because different models tokenize differently, it directly impacts both the processing and the pricing of the model

### Tokenizer Prompt Example

- Using a complex system prompt as an example of input text:
    - **Prompt Content**:

        > You're an advanced AI assistant to provide detailed, nuanced, and specific answers to complex questions. Your goal is to help users solve problems in a clear, accurate, and concise manner while maintaining a conversational friendly tone. The response should be thorough incorporating relevant background knowledge, practical examples, and any explicit context stated in the query.

    - **Tokenization Results**:
        - **Tokens**: 103
        - **Characters**: 612

### Prompt Comparison Strategy

- To observe how different prompt lengths impact token counts, the speaker uses three distinct types of prompts generated by ChatGPT:
    - High token usage
    - Balanced token usage
    - Low token usage
- **[Goal]** To demonstrate that while the fundamental nature or task of the prompts may be the same, the volume of tokens consumed can vary drastically based on the level of detail and length.

### Balanced Token Usage Example

- **Prompt Content**:

    > You are an AI assistant tasked with providing clear, accurate, and concise answers. Ensure responses include practical examples and relevant details. Tailor your explanation to the user's level of expertise using plain language and avoiding unnecessary jargon.

- **Tokenization Results**:
    - **Tokens**: 45
    - **Characters**: 262
- **[Observation]** This version significantly reduces token count compared to the high-usage prompt while still maintaining the necessary context and instructions for the AI.

### Low Token Usage Example

- **Prompt Content**:

    > Answer clearly and concisely. Use examples if needed.

- **Tokenization Results**:
    - **Tokens**: 18
- **[Note]** While this is very cheap and efficient, it may not be the ideal way to prompt an agent because it lacks the specific nuance and depth required for complex tasks.

### Prompt Comparison Results

- Comparing the three prompt types (high, balanced, and low usage) reveals how complexity impacts tokenization:
    - **High usage prompt**: Generates the most tokens
    - **Balanced usage prompt**: Generates a moderate number of tokens
    - **Low usage prompt**: Generates the fewest tokens
- **[Why this matters]** When refining prompts for AI agents, being aware of these differences helps in managing token consumption and optimizing costs.
